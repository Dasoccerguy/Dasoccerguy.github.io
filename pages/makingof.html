The Making of the Website

Inspiration

Everyone should make a website. I've owned this domain since 2015 and barely touched it at all. I certainly think I'm capable, but for some reason I've just been so, so lazy.

Methodology

Nothing is worse than getting partway through a project and realizing that you want nothing more than to scrap it all and start again from scratch. A wise coworker once told me that you are only qualified to start working on a project by the time you've finished it, and I always try to take that message to heart.

For my website, I wanted to take all the smart, correct approaches, or at least take the approaches which seem smart to me at this point. I also want to do it all myself, so I am foregoing anything like Wordpress or prebuilt CSS themes for the sake of forcing myself to learn it all. There are a few core tenets of my coding practices that I will be keeping in mind throughout this project

  • Compartmentalize everything into functions, classes, files, folders, and subfolders
  • Don't repeat myself or nearly repeat myself
  • Keep it easily human-readable, and probably even add comments
  • Look and feel must be intertwined seamlessly

    Technicals

    The domain is hosted on Bluehost, which seems dependable and reasonably cost-efficient. I purchased it in August, 2015.

    At the outset, I am using Sublime Text 3 to edit HTML, CSS, and javascript. It seems to work pretty well, and thats with me being a complete n00b and not knowing how to use any of the special features aside from context-sensitive autocomplete things. I will definitely be trying out Atom before committing to any purchases.

    It took all of a half hour of using Bluehost's embedded cPanel FTP situation connected to FileZilla before I was incredibly tired of that workflow. I am a firm believer in git, but I didn't see an easy way to get git working on Bluehost. So, I embarked on a quest to combine git and ftp locally

    A quick search led me to git-ftp, an extension which integrates ftp calls into git. Since I am a Windows pleb (for now), I had to first install Cygwin, which provides a terminal for a Linux-like experience on Windows, and then install git-ftp through the cygwin terminal. I struggled to get the directory configuration (both locally and remote) like it needed to be, and kept accidentally nesting copies of the website within itself. It was a beautiful moment when I finally saw my 'git ftp push' command work, which "pushed" the commit straight to my website.
